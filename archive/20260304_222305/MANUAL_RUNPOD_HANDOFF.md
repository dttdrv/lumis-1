# Manual RunPod Handoff (Human Operator)

This repository does not assume any RunPod access. Use this checklist manually.

## 1. Prepare local artifacts first

1. Validate identity pack:
   - `python scripts/validate_identity_pack.py --identity-dir <path>`
2. Build open corpus (dry-run first, then full):
   - `python scripts/build_open_corpus.py --config configs/dataset_mixture.yaml --dry-run --small-sample`
   - rerun without `--dry-run` for full build when ready.
3. Merge full warehouse:
   - `python scripts/merge_full_warehouse.py --config configs/dataset_mixture.yaml`
4. Validate full dataset:
   - `python scripts/validate_full_dataset.py --config configs/dataset_mixture.yaml`
5. Render final manifest:
   - `python scripts/render_dataset_manifest.py --config configs/dataset_mixture.yaml`

## 2. Upload to RunPod manually

Upload these paths to your pod volume:

- `configs/`
- `scripts/`
- `notebooks/`
- final dataset outputs (from merge/manifest steps)
- optional local source snapshots if using offline mode

## 3. Install environment in pod

1. Create venv/conda env.
2. Install:
   - `pip install -r requirements.txt -c constraints.txt`
3. Verify:
   - `python -c "import unsloth, transformers, trl, datasets; print('ok')"`

## 4. Execute notebook order on RunPod

1. `notebooks/10_validate_identity_pack.ipynb`
2. `notebooks/20_build_open_dataset_mix.ipynb`
3. `notebooks/30_merge_and_validate_full_dataset.ipynb`
4. `notebooks/40_train_sft_unsloth_qwen35_4b.ipynb`
5. `notebooks/50_train_dpo_unsloth_qwen35_4b.ipynb` (optional after SFT)
6. `notebooks/60_eval_export_smoke.ipynb`

`notebooks/00_repo_audit_and_cleanup.ipynb` is documentation/audit support and can be run earlier.

## 5. Run safety defaults first

- Start with `DRY_RUN=True` and `SMALL_SAMPLE=True`.
- Confirm paths and output directories.
- Then switch to full run settings.

## 6. Resume behavior

- SFT and DPO notebooks support resume-from-checkpoint via config cell fields.
- Keep output directories stable between restarts.
- Record run metadata JSON generated by notebooks.

## 7. Non-claims reminder

- This repo only prepares artifacts and runbooks.
- Any training/eval/export result exists only after you run those notebooks on your pod.
