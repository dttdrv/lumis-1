AUTONOMOUS SUPERVISOR LOOP
Loop until state.status is finalized or fatal:
1) Load state.
2) If targets reached and status=active, run FINALIZATION PHASE.
3) Else run CHUNK PHASE.
4) Update heartbeat.log.
5) Continue loop automatically (no user intervention).

Do not exit loop early for non-fatal errors.

CHUNK PHASE
A) Determine chunk size:
- remaining_sft = target_sft - sft_written
- remaining_pref = target_pref - pref_written
- chunk_sft = min(state.chunk_sft, remaining_sft)
- chunk_pref = min(state.chunk_pref, remaining_pref)

If chunk_sft==0 and chunk_pref==0, jump to FINALIZATION PHASE.

B) Recovery for current chunk_id=state.next_chunk:
- If pending shard files exist, validate and reuse if valid.
- If pending files are partial/corrupt, delete pending files for this chunk and regenerate.

C) Mandatory subagent topology for each chunk:
- gen_a, gen_b (generation workers)
- review_a, review_b (shard reviewers)
- review_merge (merged chunk reviewer)

D) Subagent split:
- quota_a_sft = ceil(chunk_sft/2)
- quota_b_sft = chunk_sft - quota_a_sft
- quota_a_pref = ceil(chunk_pref/2)
- quota_b_pref = chunk_pref - quota_a_pref

E) Run gen_a and gen_b in parallel using GENERATOR SUBAGENT TEMPLATE.
F) Run review_a and review_b in parallel using REVIEWER SUBAGENT TEMPLATE.
G) Merge approved shard outputs into tmp merged files.
H) Run review_merge on tmp merged files.
I) If review_merge passes, atomically commit chunk and update state.
J) If review_merge fails, perform retry strategy (below).

GENERATOR SUBAGENT TEMPLATE (inline)
ROLE
Generate one shard for one chunk.

INPUTS
- run_id
- chunk_id
- shard_id in {gen_a, gen_b}
- target_sft_shard
- target_pref_shard
- output_root

WRITES ONLY
- chunks/pending/sft_chunk_<chunk_id:04d>_<shard_id>.jsonl
- chunks/pending/pref_chunk_<chunk_id:04d>_<shard_id>.jsonl
- chunks/pending/meta_chunk_<chunk_id:04d>_<shard_id>.json

RULES
- Generate exactly target_sft_shard accepted SFT rows.
- Generate exactly target_pref_shard preference rows.
- For each accepted row, evaluate 3 candidates and keep best.
- Use rubric dimensions:
  identity_fidelity, tone, brevity, honesty, capability_accuracy,
  no_fake_tools, no_visible_cot, multilingual_consistency.
- Keep only accepted score >= 84.0.
- Preference requires chosen_score > rejected_score and margin >= 0.8.
- Allowed rejected failure modes only:
  too_long, too_vague, identity_drift, wrong_creator, wrong_name,
  fake_memory, fake_tool, fake_browsing, overconfident,
  image_hallucinating, cot_leak, off_brand_tone, multilingual_inconsistent.
- Output valid JSONL and unique IDs per shard.

SUBAGENT OUTPUT
Success:
GEN_COMPLETE run_id=<...> chunk_id=<int> shard=<gen_a|gen_b> sft=<int> pref=<int>
Failure:
GEN_PAUSED run_id=<...> chunk_id=<int> shard=<gen_a|gen_b> reason=<...>

REVIEWER SUBAGENT TEMPLATE (inline)
ROLE
Strict policy/schema reviewer.

INPUTS

- chunk_id
- review_target in {gen_a, gen_b, merge, final_sft, final_pref, final_report}
- output_root

FILE MAP

If review_target=gen_b:

2) Required schema keys exist.

REVIEWER OUTPUT

CHUNK COMMIT RULES
After review_a and review_b pass:
1) Merge approved A+B into:
   - chunks/sft_chunk_<chunk_id:04d>.tmp.jsonl
   - chunks/pref_chunk_<chunk_id:04d>.tmp.jsonl
2) Trim deterministically to exact chunk_sft/chunk_pref.
3) Validate tmp JSONL.
4) Run review_merge.
5) If review_merge passes:
   - atomically rename to:
     - chunks/sft_chunk_<chunk_id:04d>.jsonl
     - chunks/pref_chunk_<chunk_id:04d>.jsonl
   - append exact signatures to dedupe_exact.sha256.txt
   - state.sft_written += chunk_sft
   - state.pref_written += chunk_pref
   - state.next_chunk += 1
   - state.consecutive_failures = 0
   - state.updated_at_utc = now
6) If review_merge fails:
   - increase state.consecutive_failures by 1
   - retry with adaptive fallback (below)

ADAPTIVE FALLBACK (automatic)
On chunk failure:
1) Retry with regenerated shards once at same chunk size.
2) If still failing, halve chunk sizes:
   - chunk_sft = max(chunk_sft_min, floor(chunk_sft/2))
   - chunk_pref = max(chunk_pref_min, floor(chunk_pref/2))
3) Retry up to 3 times per chunk.
4) If all retries fail for this chunk:
   - write review_logs/fatal_chunk_<chunk_id:04d>.json with reasons
   - set state.status = "fatal"
   - print FATAL_ABORT and stop.

FINALIZATION PHASE
When totals reached:
1) Merge all committed chunk files into:
   - sft_dataset.jsonl
   - preference_dataset.jsonl
2) Validate merged JSONL files.
3) Build final artifacts:
   - category_manifest.json
   - language_manifest.json
   - multimodal_manifest.json
   - review_report.json
   - spot_checks.jsonl (25 samples)
   - stress_test_pack.jsonl (deterministic stress subset)
   - run_manifest.json
   - identity_policy_sheet.md (copy from lumis1_identity_codex_prompt.txt)
   - how_dataset_could_still_fail.md
4) Run final reviewer subagents:
   - final_sft
   - final_pref
   - final_report
5) If all pass:
   - set state.status = "finalized"
   - state.updated_at_utc = now
   - print FINALIZE_COMPLETE lines
6) If any fail:
   - set state.status = "fatal"
   - print FATAL_ABORT lines

EXECUTION SAFETY

- Never stop while state.status is active.
- Continue autonomous loop until finalized or fatal.
